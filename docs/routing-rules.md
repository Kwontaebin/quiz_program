# 라우팅 규칙

## 패키지
- go_router 패키지를 사용하여 라우팅을 관리한다

## 폴더 구조
- router/ 폴더를 별도로 생성하여 라우팅 관련 파일을 관리한다
- router/app_router.dart: 메인 라우터 설정 파일
- router/routes.dart: 라우트 경로 상수 정의 (선택)

## 라우트 경로 정의
- 모든 라우트 경로는 상수로 정의하여 오타를 방지한다
- 경로명은 소문자와 하이픈(-)을 사용한다 (예: /user-profile)

## 화면 전환 방식
- context.go(): 현재 화면을 대체하여 이동한다 (히스토리 교체)
- context.push(): 새 화면을 스택에 추가하여 이동한다 (뒤로가기 가능)
- context.pop(): 이전 화면으로 돌아간다
- context.pushReplacement(): 현재 화면을 새 화면으로 교체한다

## 화면 간 데이터 전달
- 경로 파라미터: URL에 포함되는 필수 데이터 (예: /user/:id)
- 쿼리 파라미터: URL에 포함되는 선택 데이터 (예: /search?keyword=flutter)
- extra: 복잡한 객체를 전달할 때 사용한다

## 인증 및 가드
- 로그인 여부에 따라 접근을 제한하는 redirect 로직을 라우터에 구현한다
- 비로그인 사용자가 보호된 페이지 접근 시 로그인 화면으로 리다이렉트한다

## 주의사항
- View에서 직접 Navigator를 사용하지 않고 go_router의 context 확장 메서드를 사용한다
- 딥링크가 필요한 경우 라우트 경로를 명확하게 설계한다
- 중첩 라우팅이 필요한 경우 ShellRoute를 활용한다
